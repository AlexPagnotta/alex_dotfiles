- In all interactions and commit messages, be extremely concise and to the point, sacrifice grammar for the sake of concision.

## Pull Requests

When creating pull requests, follow these instructions:

- *Title*: Use the format `<type>: <description>` for the PR title, where `<type>` is one of `feat`, `fix` or `chore`, and `<description>` is a concise description of the pull request's purpose. For example: `feat: add login page`.

- *Description*: In the PR description, add a "Goals" section where you list the specific goals of the pull request in bullet points, followed by a link to the ticket if procided. For example:

```
## Goals

- Add a login page to the application.
- Add a logout button to the navigation bar.

## Ticket
[Ticket #123](https://example.com/ticket/123)s
```

## Git

- When creating commits, use the format `<type>: <description>`, where `<type>` is one of `feat`, `fix` or `chore`, and `<description>` is a concise description of the commit's purpose. For example: `feat: add login page`.

- When creating branches, use format  `<type>/<description>`, where `<type>` is one of `feature`, `fix` or `chore` , and `<description>` is a concise description of the branch's purpose, using hyphens to separate words. For example: `feature/add-login-page`.


## Plans

- Make the plan extremely concise, Sacrifice grammars for the sake of concision.
- At the end of each plan, add a "Unresolved Questions" section, where you list all unresolved questions, if any.
